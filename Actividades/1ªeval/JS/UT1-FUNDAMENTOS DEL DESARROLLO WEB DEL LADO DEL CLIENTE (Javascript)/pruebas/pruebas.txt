LA DIFERENCIA ENTRE EL USO DE VAR O NO PARA DECLARAR VARIABLES EN JAVASCRIPT (http://librosweb.es/libro/javascript/capitulo_4/ambito_de_las_variables.html)

El ámbito de una variable (llamado "scope" en inglés) es la zona del programa en la que se define la variable. JavaScript define dos ámbitos para las variables: global y local.
Si estas declarandola en el ambito global (global scope) no hay ninguna diferencia, puedes hacerlo de las dos maneras.
Si estas declarandola dentro de una función, si hay una gran diferencia. Si usas var estas creando una variable de ambito local a la función, es decir solo existirá dentro de esa función. Sin embargo, si no usas var, javascript buscara en el ambito local de la función, e ira subiendo de nivel hasta el ambito global, momento en el cual si no la ha encontrado, la creará en el ambito global.
Como caso particular, si solo estás declarando la variable, pero no asignandola, siempre tendrás que usar var.

var a =1;
b =2;

    var a = 3;// Local
    c =2;// Global

        tres =3;// Global
        }())
